# tiled-to-shapes

Converts Tiled TSX tileset files into Rust code that constructs a `TerrainTileSet` with baked vector shapes.

## How It Works

```
TSX file + tilesheet PNG
        │
        ├─ Parse Wang set definitions (corner type only)
        ├─ Extract tile sub-images from the PNG
        ├─ Vectorize each tile via img-to-shape
        ├─ Normalize shapes to [0,1]² tile space
        └─ Generate a Rust source file
```

The output is a `.rs` file containing a `pub fn tileset() -> TerrainTileSet` function with all tile shapes baked in as path commands.

## Getting Started

### 1. Prepare Your TSX File

Create or export a Tiled tileset (`.tsx`) with **corner-type Wang sets**. The tool ignores edge/mixed sets.

Place your TSX and its referenced PNG **in the same directory**:

```
assets/
  terrain.tsx
  terrain.png
```

The TSX should reference the PNG with a **relative path** in the `<image source="...">` attribute.

### 2. Add Wang Set Properties (or Scaffold Them)

Each corner Wang set needs these custom properties in Tiled's tileset editor:

| Property             | Type   | Default    | Meaning                              |
|----------------------|--------|------------|--------------------------------------|
| `passability`        | string | `passable` | `passable`, `solid`, or `difficult`  |
| `priority`           | int    | `0`        | Render order (higher = on top)       |
| `hue_shift_max`      | float  | `5.0`      | Color variation range (degrees)      |
| `brightness_shift_max` | float | `0.05`   | Brightness variation range           |

**Shortcut:** run with `--scaffold` to auto-add missing defaults:

```sh
cargo run -p tiled-to-shapes -- assets/terrain.tsx --scaffold
```

### 3. Run the Converter

```sh
# Print generated code to stdout
cargo run -p tiled-to-shapes -- assets/terrain.tsx

# Write to a directory
cargo run -p tiled-to-shapes -- assets/terrain.tsx -o src/terrain/generated/

# Custom function name
cargo run -p tiled-to-shapes -- assets/terrain.tsx -o src/terrain/generated/ --fn-name my_tileset

# Dry run (parse + validate only, no output)
cargo run -p tiled-to-shapes -- assets/terrain.tsx --dry-run
```

### 4. Use the Generated Code

The output file (e.g. `tileset.rs`) looks like:

```rust
// Generated by tiled-to-shapes from terrain.tsx
// Do not edit manually.

use std::collections::BTreeMap;
use engine_render::prelude::PathCommand;
use glam::Vec2;
use terrain::prelude::*;

pub fn tileset() -> TerrainTileSet {
    TerrainTileSet {
        tiles: BTreeMap::from([
            ("grass".to_owned(), grass()),
        ]),
        adjacency_rules: Vec::new(),
    }
}
// ...
```

Include it in your crate:

```rust
// src/terrain/mod.rs
mod generated;
pub use generated::tileset;
```

Then wire the tileset into the game wherever `TerrainTileSet` is consumed.

## TSX Format Requirements

- Tiled version 1.10+ format (XML)
- `<tileset tilewidth="…" tileheight="…" columns="…">`
- `<image source="relative/path/to/sheet.png">`
- `<wangsets>` with at least one `<wangset type="corner">`
- Each `<wangtile tileid="…" wangid="…">` entry maps a tile to a corner bitmask

**Only corner-type Wang sets are processed.** Edge and mixed sets are skipped with a warning.

## CLI Reference

```
tiled-to-shapes <input.tsx> [OPTIONS]

Arguments:
  <input.tsx>         Path to Tiled TSX tileset file

Options:
  -o, --output DIR    Output directory (default: stdout)
  --scaffold          Write missing default properties back into the TSX
  --fn-name NAME      Generated function name (default: tileset)
  --dry-run           Parse and validate only, no output
  --help              Show this help
```

## Tile Resolution

The converter maps Tiled's 8-value wangid to the 4-corner bitmask used by the terrain system:

```
wangid: "edge0, corner0, edge1, corner1, edge2, corner2, edge3, corner3"
                  NE              SE              SW              NW

corner16 bitmask: NE=1, SE=2, SW=4, NW=8
```

A corner value equal to the Wang set's terrain color index (default: 1) means "this terrain is present."

The 16 possible bitmasks are grouped into 5 `TilePattern`s:

| Pattern       | Bitmasks         | Canonical |
|---------------|------------------|-----------|
| `Solid`       | 0, 15            | 15        |
| `OuterCorner` | 1, 2, 4, 8       | 1         |
| `Edge`        | 3, 6, 9, 12      | 3         |
| `Diagonal`    | 5, 10            | 5         |
| `InnerCorner` | 7, 11, 13, 14    | 7         |

This crate lives in `tools/` which is a **separate Cargo workspace** from the main project. Build and test with:
